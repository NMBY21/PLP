Introduction

This page is part of an HTML5 assignment focused on writing clean, accessible, and SEO-friendly web pages using only semantic HTML.

Semantic HTML5 Structure

Why Semantic Tags Matter

Semantic elements like <header>, <main>, <article>, and <footer> help describe the content's purpose. They improve readability for both humans and search engines.

Accessibility Best Practices

Screen Reader Support

Using landmarks such as <nav>, <main>, and form labels ensures screen readers can navigate the page effectively.

Forms with Labels

Always associate <label> elements with their respective inputs for better accessibility.

SEO-Friendly Content

Use of Headings

Headings (h1 to h6) should follow a clear hierarchy to make the structure understandable for crawlers and users alike.

Metadata & Keywords

Including relevant metadata like description, author, and keywords in the <head> improves SEO visibility.
